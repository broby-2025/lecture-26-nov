# Laga en "trasig" webbsida

I veckans övning får ni arbeta med en webbsida där olika typer av fel har smugit sig in. Tanken är att ni ska undersöka sidan på samma sätt som ni senare kommer att granska och optimera egna projekt. Fokuset ligger alltså inte på att “gissa rätt” utan på att använda rätt verktyg och strategier för att systematiskt hitta vad som behöver lagas.

Fundera särskilt på hur ni kan:

- använda webbläsarens utvecklingsverktyg för att identifiera brutna element, felmeddelanden och oväntade DOM-strukturer  
- läsa HTML och CSS med “felsökningsglasögon” och leta efter mönster som brukar orsaka problem  
- inspektera layouten med hjälp av verktyg som låter er visualisera boxmodellen, grid- och flex-strukturer  
- utnyttja konsolen för att upptäcka varningar, fel och ledtrådar till vad som inte beter sig som det ska  
- jämföra förväntat beteende (skissen, designen eller en fungerande version) med vad sidan faktiskt gör  
- arbeta metodiskt: börja med de mest grundläggande delarna (struktur och syntax) innan ni går vidare till detaljer  

Ni behöver inte lösa allt på en gång. Målet är att träna på att se vad som inte stämmer, prova en hypotes i taget och hela tiden använda verktygen för att bekräfta eller avfärda det ni tror är fel.

### **Bonusuppgift**
Ta reda på varför knapparna **“Ta bort”** inte tar bort plantorna.  
Fokusera på att:

- kontrollera om det finns något JavaScript som ska hantera klick  
- se hur knapparna är strukturerade i DOM:en  
- undersöka om det finns fel i konsolen som stoppar funktionen  
- fundera på vad som behövs för att ett klick faktiskt ska ändra sidans innehåll

Målet är inte att bygga en avancerad lösning, utan att förstå *varför* det inte fungerar just nu.
